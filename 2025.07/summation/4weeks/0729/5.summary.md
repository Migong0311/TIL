# 개념 요약

| 개념               | 설명                                                                 | 예시                                              |
|--------------------|----------------------------------------------------------------------|---------------------------------------------------|
| 딕셔너리 (Dictionary) | 고유한 키-값 쌍을 저장하는 비시퀀스 컬렉션                            | `{"name": "Alice", "age": 25}`                   |
| 세트 (Set)          | 중복을 허용하지 않는 정렬되지 않은 컬렉션                             | `{0, 1, 2}`, `union({1, 3}) → {0, 1, 2, 3}`        |
| 해시 테이블 (Hash Table) | 키를 해시 값으로 변환해 배열 인덱스로 사용해 빠른 검색·삽입·삭제를 지원 | 책 제목(키)→책위치(해시→인덱스)→책장 번호 이동     |
| hashable (해시 가능) | 내부 값이 변하지 않으며 해시 값을 유지하는 객체만 dict, set 요소로 사용 가능 | `hash("a")`, `hash((1,2,3)) → 정상`, `hash([1,2,3]) → TypeError` |
| 집합 연산 (Union / Intersection 등) | 두 세트의 합집합·교집합·차집합 등을 통해 중복 제거와 관계 분석 수행 | `set1.union(set2) → {0,1,2,3,4,5,7,9}`            |
| `dict.get()`        | 키가 없을 때 기본값을 반환해 안정하게 조회할 수 있는 메서드            | `person.get("country", "Unknown")`               |
| 키 (Key)            | 값에 접근하기 위한 고유 식별자 – 인덱스 대신 사용해 가독성과 검색 속도 향상 | `phonebook["민수"]` → 번호 조회                  |
| EBNF 메타기호 `[]`  | 공식 문서에서 선택적 매개변수를 나타내는 표기법 (대괄호 안은 선택)      | `pop(key[, default])` → `default`는 선택 가능    |


# #️⃣ ref()

## 🔵 반응형 상태 (Reactive State)

### 📌 **ref() = 반응형 상태(데이터)를 선언하는 함수**

일반 JavaScript 변수를
→ **Vue가 변화(변경)를 감지할 수 있는 반응형 객체로 변환**해 줍니다.

즉, 컴포넌트 내부에서 변하는 값(숫자, 문자열, input 값 등)의 상태를 추적하고 관리하기 위해 사용합니다.

---

# 🔵 ref 함수 (1/2)

### ✔ ref는 내부적으로 `.value` 속성을 가진 “ref 객체”로 wrapping(랩핑)함

* ref()로 만든 변수는 원시값이 아닌 **객체 형태**가 되며
  실제 값은 `refVar.value` 에 저장됩니다.

### ✔ ref로 선언한 값이 변경되면

→ 해당 값을 사용하는 템플릿은 자동으로 업데이트(render)

### ✔ ref는 어떤 타입도 인자로 받을 수 있음

숫자, 문자열, boolean, 객체, 배열 등 모두 가능

---

### 📘 예시 코드

```html
<script>
  const { createApp, ref } = Vue

  const app = createApp({
    setup() {
      const message = ref('Hello vue!')

      console.log(message)          // ref 객체
      console.log(message.value)    // Hello vue!

      return {}
    }
  })
</script>
```

🔎 **ref 객체란?**
→ 값(value)을 `.value` 안에 감싸서 저장하는 “반응형 껍데기(wrapping)” 객체

---

# 🔵 ref 함수 (2/2)

### ✔ 템플릿에서 ref 값을 사용하려면

→ setup() 함수에서 **선언 + return** 해야 템플릿에서 접근 가능

### ✔ 템플릿에서는 `.value`를 작성할 필요 없음

→ Vue가 자동으로 언래핑(unwrapping) 해줌
→ 즉, 템플릿에서는 `{{ message }}` 라고 바로 사용 가능

---

### 📘 예시 코드

#### JS (setup 내부)

```js
const app = createApp({
  setup() {
    const message = ref('Hello vue!')

    return {
      message
    }
  }
})
```

#### HTML 템플릿

```html
<div id="app">
  <h1>{{ message }}</h1>
</div>
```

✔ JS에서는 `message.value`
✔ 템플릿에서는 `{{ message }}`
→ Vue가 자동 처리!

---

# 🎯 결론

### ref 함수란?

> **ref()는 "반응형을 가지는 참조 변수를 만드는 함수"**
> ref === Reactive Reference

* ref는 값을 `.value` 내부에 보관
* JS에서는 `.value`로 접근
* 템플릿에서는 언래핑되어 `.value` 없이 바로 사용 가능
* 상태가 바뀌면 화면도 자동으로 렌더링

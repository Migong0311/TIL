# 3.Vue_Application

## 📦 Component

### 🔹 재사용 가능한 코드 블록

* UI를 독립적이고 재사용 가능한 부분으로 분리하고 각 부분을 개별적으로 다를 수 있음
* 자연스럽게 애플리케이션은 중첩된 Component의 트리 형태로 구성됨

---

# 🌐 Vue를 사용하는 방법

1. **CDN 방식**
2. **NPM 설치 방식**

   * 이후 수업은 CDN 방식 기준으로 진행됨

---

# 🟦 Vue Application 생성하기 (1/6)

### ✔ Vue 사용을 위한 CDN 작성

```html
<!-- vue-instance.html -->
<div id="app"></div>

<script src="https://unpkg.com/vue@3/dist/vue.global.js"></script>

<script>
  const { createApp } = Vue

  const app = createApp({
    setup() {}
  })

  app.mount('#app')
</script>
```

---

# 🟦 Vue Application 생성하기 (2/6)

### ✔ Application instance란?

* CDN에서 Vue를 사용할 경우 **전역 Vue 객체(Vue)** 를 불러올 수 있음
* 구조분해 할당 문법으로 Vue 객체의 createApp 함수를 불러옴

```js
const { createApp } = Vue
```

---

# 🟦 Vue Application 생성하기 (3/6)

### ✔ Application instance 생성

* 모든 Vue 애플리케이션은 `createApp()` 함수를 호출해 새로운 Application instance를 생성하는 것부터 시작함

```js
const app = createApp({
  setup() {}
})
```

---

# 🟦 Vue Application 생성하기 (4/6)

### ✔ Root Component

* `createApp()` 함수에는 객체(컴포넌트)가 전달됨
* 모든 App은 다른 컴포넌트를 하위로 포함할 수 있는 **Root(최상위) 컴포넌트**가 필요
  (현재는 단일 컴포넌트를 Root로 사용)

```js
const app = createApp({
  setup() {}
})
```

---

# 🟦 Vue Application 생성하기 (5/6)

### ✔ Mounting the App (앱 연결)

* HTML 요소에 Vue Application instance를 "탑재(연결)"함
* 한 개의 인스턴스에 대해 `mount()` 는 **한 번만 호출 가능**

```html
<div id="app"></div>

<script>
  const app = createApp({
    setup() {}
  })

  app.mount('#app')
</script>
```

---

# 🟦 Vue Application 생성하기 (6/6)

### ✔ setup() 함수

* 컴포넌트가 동작하기 전에 미리 준비하는 **시작점 / 초기 설정용 함수**
* setup 안에서

  * 데이터를 정의하거나
  * 화면에 표시할 값을 계산하거나
  * 각종 로직(함수)을 준비할 수 있음
* setup()에서 정의한 값들은 이후 템플릿이나 컴포넌트의 다른 부분에서 바로 사용 가능함

```js
setup() {
  // 데이터, 로직 준비하는 자리
}
```
